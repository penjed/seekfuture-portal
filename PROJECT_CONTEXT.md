# Project Context

## Product

- Position: 给小公司做低成本、好用、能快速上线的信息化 + AI 定制系统。
- Target user: 还在用微信、Excel、人工沟通管理业务的小公司；他们希望提效和增强竞争力，但不太愿意购买昂贵、复杂、绑定深的软件公司产品。
- Current priority: 优化 `ai_enterprise_ad_webpage.html`，把页面从通用 AI 数字化宣传页升级为更清晰、更可信、更能转化的个人定制服务落地页。

## Core Offer

- 从客户最头疼的一个业务流程开始，用小程序、管理系统和 AI 做一个能用、便宜、见效快的小系统。
- 不强调“什么软件都能做”，改为强调“低成本把微信、Excel、人工流程升级成可用的信息化 + AI 系统”。
- 主推组合方案，而不是单独罗列技术菜单。

## Target Solutions

- 业务流程信息化：预约、订单、售后、工单、客户管理。
- 企业知识库 + AI 助手：资料查询、员工培训、客服高频问答、流程指导。
- 小程序 + 管理后台 + AI 自动化：客户入口、内部管理、自动回复、统计、提醒和流程推动。

## Conversion Goal

- 页面先让访客意识到自己的业务需要信息化和 AI 提效。
- 然后引导他们联系做一次低成本、低门槛的业务流程诊断。
- Primary CTA: 免费诊断你的业务流程。

## Proof Points

- 做过售后工单系统：客户报修、派单、进度跟踪更清楚。
- 做过内部知识库：员工不用反复问资料和流程。
- 做过小程序预约系统：客户在线预约，减少人工沟通。

## Principles

- Product: 聚焦小公司真实业务问题，从一个流程开始，不做大而全的空泛承诺。
- Design: 简洁高级、可信、克制，避免模板感和过度炫技。
- Engineering: 优先快速上线和可维护；除非必要，避免复杂架构。

## Decisions

- 2026-07-15: 页面定位从“AI 企业数字化解决方案”调整为“帮小公司从最痛的一个流程开始，低成本升级信息化和 AI”。原因是原定位过泛，客户难以快速对号入座；新定位更可信、更适合个人前期获客。Revisit when: 有明确行业客户或真实案例后，可以进一步聚焦垂直行业。

## Non-Goals

- 不把页面写成“什么软件都能做”的万能外包宣传。
- 不假装已有大型成功案例。
- 不优先做复杂 SaaS 平台叙事。
